status=3 的词条列表，无需独立库表——整个系统仅一张单词主表，审核池是同一个表按 status 筛选的视图。审核池为系统统一入口，无论单词来自单词库侧上传还是资源库侧上传，均在此处完成人工审核。审核通过后的下游处理：审核仅变更词条 status 状态（3→0），与频次无关。频次由资源库侧文件上传识别时独立直接累加至 total_frequency 及 tag_frequency_json，不等审核；单词库侧上传不产生频次变更。审核删除时，若为资源库侧新增单词（频次已在上传时累加），同步回滚频次（扣减 total_frequency 及 tag_frequency_json，删除对应 file_word_frequency 记录）。频次与状态解耦：频次（total_frequency）由文件上传/删除自动累计/扣减，不改变单词的 status 状态。单词是否可被用户学习，由前端查询时双重判断：status=1（已上架）AND total_frequency > 0。频次为零的前端过滤：total_frequency=0 的单词，即使 status=1（已上架），前端查询时因 total_frequency > 0 条件不满足而不返回，用户不可见、不可学习。管理员在后台仍可见该词条，可正常执行上架/下架操作。 频次归零后标签联动清空：当某单词的 total_frequency 因文件删除/失效操作归零时，该单词的 stage_label（绑定标签）字段同步清空。设计依据：标签来自资源库教材上传时的学段特征自动分配，频次归零意味着所有支撑该单词的教材文件均已删除/失效，对应的标签绑定随之失去依据，前端自然也因 total_frequency > 0 条件不满足而不可见。前端查询条件（学习端）：查询可用单词时，SQL 条件为 WHERE status=1 AND total_frequency > 0，确保只有已上架且有素材支撑的单词才返回给学习者。已删除（is_delete=1）：软删除，数据库永久保留数据但标记删除，前端不可见、不可恢复至正常状态（仅可通过数据库层面恢复）4 功能详细需求（核心模块）4.1 重写：4引擎大模型增强方案）本系统对上传文件中识别到的所有英文单词执行形态归集判定。核心目标：区分规则变形（不建新词，频次归集到主词）与不规则变形（独立建词，独立频次，建立同源关联），杜绝同源单词拆分统计导致的数据失真。系统采用「规则表驱动 + 大模型增强」的混合架构，确保高频路径毫秒级、低频路径一次AI、极低频率人工兜底。4.5.7.1 四引擎归集流水线AI 提取到一个英文单词后，按以下顺序依次经过四个引擎处理：第一引擎：单词主表查重查询单词主表 word 表，WHERE word = 当前词（忽略大小写）。若已入库，则直接累加频次（total_frequency + tag_frequency_json），流程结束。该引擎处理已入库单词的复用场景，0次额外调用。第二引擎：算法规则匹配针对未命中主表的单词，通过词尾模式匹配 + 还原算法判断是否为规则变形。支持以下变形类型的自动还原：• 动词三单：s/es/ies 词尾 → 还原为原形（如 takes→take, goes→go, studies→study）• 现在分词：ing 词尾 → 还原为原形（如 taking→take, running→run）• 规则过去式/过去分词：ed/d/ied 词尾 → 还原为原形（如 walked→walk, liked→like, studied→study）• 名词复数：s/es/ies 词尾 → 还原为原形（如 books→book, boxes→box, babies→baby）• 比较级/最高级：er/ier、est/iest 词尾 → 还原为原形（如 faster→fast, easiest→easy）还原后必须校验候选主词是否已在单词主表中存在。主词已入库 → 确认为规则变形，频次全量累加到主词，不新建词条。主词未入库 → 规则匹配失败，继续进入第三引擎。第三引擎：不规则变形查表查询 word_inflection_rule 变形规则表，SELECT lemma, type FROM word_inflection_rule WHERE inflected = 当前词。命中 → 获取原形主词（lemma）和变形类型（type），确认主词已在主表中 → 判定为不规则变形，独立建词条（独立频次），并在 word_relate 字段建立与主词的同源关联。未命中 → 进入第四引擎。第四引擎：大模型兜底判断前三引擎均未命中的单词，调用大语言模型（LLM）判断其原形和变形类型。Prompt 设计原则：• 输入：单个英文单词• 输出：严格JSON格式 {"lemma": "原形词", "type": "类型枚举", "confidence": 0.0~1.0}• type枚举：regular（规则变形）/ irregular_past / irregular_plural / irregular_comp / variant / base（原形）/ unknown（不确定）大模型返回后的处理：• type != "unknown" 且 confidence ≥ 0.85：自动将结果写入 word_inflection_rule 表（后续自动命中第三引擎），并执行归集操作• type != "unknown" 但 0.5 ≤ confidence < 0.85：写入规则表但标记 priority=2（低置信），本次仍归集，审核池标记「归集待确认」• type = "unknown" 或 confidence < 0.5：不写入规则表，按独立新词处理（status=3），标记「归集待确认」，进入审核池等待人工判断大模型仅在首次遇到未命中单词时调用一次，调用后将结果写入规则表，后续同一单词直接命中第三引擎，不再重复调用大模型。4.5.7.2 规则变形处理逻辑判定为规则变形的单词：不创建新的独立词条。将该变形词出现的频次全部累加至原形主词的 total_frequency 和对应标签的 tag_frequency_json。在 file_word_frequency 溯源表中记录原始识别形态（source_form 字段保留变形原文），确保频次溯源可追溯到原始素材中的具体形态。单词排序：后台列表支持按频次倒序/正序排序，快速筛选高频核心词、低频冷门词学习推荐：支撑前端智能推荐逻辑，频次越高优先级越高，学生优先学习学段高频必考单词标签词库优化：基于分标签频次，自动筛选各标签核心高频词汇，辅助精准组词库数据统计：支持按学段统计高频词TOP榜单，为词库迭代、教学素材优化提供数据支撑4.6 交互规范单词黑名单管理黑名单用于存放管理员从审核池或单词管理中移入的单词（status=4）。黑名单单词不再出现在审核池和正常单词列表中，但其频次正常累计不受影响。黑名单的核心价值在于：当相同单词再次通过文件上传进入系统时，将被自动拦截，不再出现在审核池中，从而减少重复审核的工作量。4.7.1 黑名单移入规则审核池移入：管理员在审核池点击「删除」按钮，单词 status 从 3（审核中）变更为 4（黑名单），从审核池列表移除。操作前弹出二次确认提示框，确认后方可执行。单词管理移入：管理员在单词管理页点击「删除」按钮，单词 status 从当前状态变更为 4（黑名单），从单词管理默认列表移除。操作前弹出二次确认提示框，确认后方可执行。单词管理状态筛选器中可单独选择「黑名单」查看。频次处理：移入黑名单后，单词的 total_frequency 和 tag_frequency_json 保持不变，后续资源库上传中若再次出现该单词，频次正常累加。即黑名单仅改变状态，不影响频次统计。移入原因记录：每次移入黑名单时记录操作人和操作时间，写入 operation_log 表，日志类型为「移入黑名单」或「批量移入黑名单」。4.7.2 黑名单列表页黑名单页面位于左侧导航「系统设置」分组下，展示所有 status=4 的单词列表。列表字段包括：ID、单词、音标、释义、标签、总频次、移入时间、来源（手动新增/资源库上传/单词库上传）、操作（恢复）。支持关键字搜索（按单词英文名模糊匹配），支持分页（默认每页20条），分页组件与全站统一。黑名单页面不提供「删除」操作，仅提供「恢复」操作。4.7.3 黑名单恢复规则管理员可在黑名单页面点击「恢复」按钮，将单词从黑名单恢复至审核池。操作前弹出二次确认提示框，确认后方可执行。恢复后单词 status 从 4（黑名单）变更为 3（审核中），重新进入审核池，需管理员重新审核（通过→待上架→上架）。恢复后频次数据不变，无需回滚或重新计算。恢复操作写入 operation_log 表，日志类型为「黑名单恢复」。4.7.4 黑名单拦截规则当通过单词库上传（§4.1.2）或资源库上传（§4.6）新增单词时，系统在查重阶段同时检查黑名单：若待新增的单词已存在于黑名单中（status=4），则跳过该单词，不生成新的审核记录，不进入审核池。拦截行为对上传流程透明：上传方不会感知到单词被拦截，仅统计时排除黑名单单词。拦截不产生操作日志（属于系统自动行为，非人工操作）。黑名单单词的频次在后续上传中正常累加：拦截的是「新生成审核记录」这一行为，而非「频次累加」。频次累加由资源库文件单词频次关联逻辑独立处理，不受黑名单状态影响。4.7.5 黑名单与状态枚举黑名单状态值 status=4 属于单词生命周期枚举的一部分，与 0（待上架）、1（已上架）、2（已下架）、3（审核中）共用同一 status 字段。黑名单状态转换规则如下：• 审核中（3）→ 黑名单（4）：管理员审核驳回• 待上架/已上架/已下架（0/1/2）→ 黑名单（4）：管理员在单词管理页删除• 黑名单（4）→ 审核中（3）：管理员在黑名单页恢复• 黑名单（4）→ 其他状态：暂不支持直接流转，必须经由审核中（3）恢复后重新走正常生命周期。4.8 短语库管理短语库是独立于单词库的核心模块，存放 AI 从资源库上传文件中识别出的短语和语法句型。短语与单词共用标签体系和频次计算逻辑，但表结构更简洁。4.8.1 短语识别与来源识别范围：动词短语（如 take off）、固定搭配（如 at the same time）、常用句型（如 it is adj. for sb
