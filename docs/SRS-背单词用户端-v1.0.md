## 1. 引言

### 1.1 编写目的

本文档定义「大白陪你背单词」用户端的完整软件需求规格，作为产品、设计、开发、测试各角色的统一依据。

### 1.2 适用范围

覆盖微信小程序、Android、iOS 三端用户端。不包含管理后台。

### 1.3 术语与缩略语

| 术语 | 说明 |
|------|------|
| 学习组件 | 单词学习过程中的最小交互单元，如读单词、拼单词、英译汉等 |
| 新学 | 首次接触新单词的学习环节 |
| 复习 | 对已学单词的间隔重复复习环节 |
| 直选 | 从若干选项中直接选择正确答案的题型 |
| 全拼 | 通过字母键盘逐字母拼写出单词的题型 |
| 拼读 | 结合语音朗读与字母拼写的混合题型 |
| 录音评价 | 用户朗读单词后，系统通过语音识别给出发音评分 |
| 打卡 | 每日完成学习任务后的签到行为 |
| 词库 | 按年级或考试类型组织的单词集合 |

### 1.4 参考文献

- SM-2 间隔重复算法 (SuperMemo)
- 《微信小程序设计指南》
- Material Design 3 规范
- iOS Human Interface Guidelines

## 2. 整体描述

### 2.1 产品视角

「大白陪你背单词」是一款面向 K12 学生及英语学习者的智能背单词应用。以 IP 角色"大白"为伴学形象，融合语音识别评分、多维度学习组件（读、拼、译、听）、间隔重复算法，提供趣味化、科学化的单词学习体验。

**核心价值主张**：

1. **多维学习**：读、拼、译、听四大维度覆盖，不只是看和记
2. **语音评测**：AI 录音评价，帮助用户纠正发音
3. **伴学 IP**：大白形象陪伴，学习过程不枯燥
4. **科学复习**：基于遗忘曲线的间隔重复算法

### 2.2 用户特征

| 用户类型 | 特征描述 | 典型场景 |
|---------|---------|---------|
| K12 学生 | 按年级选词库，有明确的学习进度 | 每日按计划新学+复习，打卡坚持 |
| 备考用户 | 有考试目标（四六级、考研等） | 针对性选择词库，强化训练 |
| 兴趣用户 | 无明确考试目标，以提升词汇量为目的 | 利用碎片时间学习 |

### 2.3 运行环境

| 平台 | 要求 |
|------|------|
| 微信小程序 | 基础库 >= 3.0.0，微信 >= 8.0 |
| Android | 最低 Android 8.0（API 26），目标 Android 14 |
| iOS | 最低 iOS 14.0，目标 iOS 17.0 |

### 2.4 设计与实现约束

- 跨平台框架优先考虑 Taro / uni-app
- 后端接口 RESTful API，JSON 格式，HTTPS 传输
- 语音评测需要调用后端 AI 接口，对网络有一定依赖
- 核心学习流程支持离线使用（离线时跳过录音评测）

### 2.5 假设与依赖

- 用户设备有稳定网络（首次加载、同步、录音评测时）
- 词库数据由管理后台维护
- 发音音频文件托管于 CDN
- 语音评测服务由第三方 AI 引擎提供

| 编号 | FR-AUTH-001 |
|------|------------|
| 功能名称 | 微信一键登录 |
| 优先级 | P0 |

**界面描述**（参考效果图：01登录）：

- 页面顶部：应用 Logo + "大白陪你背单词" 标题

- 页面底部：绿色「微信一键登录」按钮（圆角，宽度撑满两侧留边距）

**交互流程**：

1. 用户点击「微信一键登录」

6. 老用户直接进入首页

**备选流程**：
- 3a. 用户拒绝授权 → 提示「需要授权才能使用」，留在登录页

##### 3.4.3.1 读单词（含录音评价）

| 编号 | FR-COMP-001 |
|------|------------|
| 功能名称 | 读单词 |
| 优先级 | P0 |

**界面描述**（参考效果图：01_1~01_7 读单词系列）：

**初始态**：
- 顶部：进度指示 + 关闭按钮
- 中部：单词（大字号）+ 音标 + 发音按钮
- 底部：「按住录音」按钮（圆形麦克风按钮，灰色常态）

**录音态**：
- 用户按住录音按钮 → 按钮变为红色脉冲动画，显示录音中状态
- 同时播放标准发音供用户跟读
- 最短录音时长 3 秒

**录音少于 3 秒提示态**：
- 弹出 Toast 提示："录音时间太短，请再试一次"
- 停止录音，回到初始态

**正在分析态**：
- 录音完成后 → 显示"正在分析..."动画（loading 指示器）
- 等待后端语音评测接口返回

**分析超时提示态**：
- 如果评测接口超时（>10秒）→ Toast 提示："分析超时，请重试"
- 回到初始态

**录音评价结果态（高分）**：
- 显示评价分数（如 90 分，绿色大字号）
- 显示评价文案："太棒了！"/ "发音很标准！"
- 绿色对勾图标
- 「继续」按钮

**录音评价结果态（低分）**：
- 显示评价分数（如 55 分，橙色/红色）
- 显示评价文案："继续加油"/ "再试一次"
- 显示标准音标和发音提示
- 「再试一次」和「继续」按钮

**交互流程**：
1. 展示单词和音标 → 自动播放一次标准发音
2. 用户按住录音按钮朗读
3. 录音时长 < 3 秒 → 提示太短，重新录音
4. 录音完成 → 发送到语音评测接口
5. 等待分析结果（loading 态）
6. 显示评分结果 → 用户点击「继续」进入下一个学习组件

**业务规则**：
- 录音最短 3 秒
- 评测超时阈值 10 秒
- 评分范围 0-100
- 高分阈值 >= 80，低分 < 60

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 首页加载时间 | <= 2 秒（含本地缓存） |
| 学习组件切换 | <= 300ms |
| 录音评测响应 | <= 5 秒（正常网络） |
| 音频播放延迟 | <= 500ms |
| 离线学习可用 | 核心学习组件离线可用（录音评测除外） |

### 4.2 可用性需求

- 学习流程操作步数最简化，核心路径 ≤ 3 步进入学习
- 按钮尺寸 >= 44px（符合移动端触控标准）
- 文字清晰可读，字号 >= 14px（正文）
- 色彩对比度符合 WCAG AA 标准

### 4.3 安全性需求

- 所有 API 请求使用 HTTPS 加密
- 用户敏感信息（手机号）脱敏展示
- 登录态 token 有效期 7 天，支持静默续期
- 语音数据不上传至第三方（仅发送至评测引擎处理后销毁）

### 4.4 可靠性需求

- 学习进度实时保存，异常退出不丢失
- 网络中断时自动切换离线模式
- 语音评测接口超时自动降级（跳过评测环节）

### 4.5 兼容性需求

- 微信小程序兼容 iOS / Android 微信客户端
- Android 8.0 ~ Android 14 全覆盖
- iOS 14.0 ~ iOS 17.0 全覆盖
- 适配屏幕尺寸：4.7 寸 ~ 6.9 寸

- userId, date, wordsLearned, wordsReviewed, streakDays
