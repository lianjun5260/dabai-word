# 2026-06-20 工作日志

## P32 LC-06 字母行DOM差量更新修复
- 问题：移动端Chrome上，字母下划线行显示顺序与键盘输入顺序相反（输入abc→显示cba）
- 根因：renderSpellLine() 每次 oninput 都用 innerHTML 重建整个flex容器，移动端Chrome flex渲染存在时序问题
- 修复：改为DOM差量更新
  - loadLc06() 中创建 cell DOM 节点并存入 lc06State._cells[]
  - renderSpellLine() 只更新每个 cell 的 textContent 和 className，不重建容器
- 不影响其他组件（renderSpellLine 仅 LC-06 使用）
