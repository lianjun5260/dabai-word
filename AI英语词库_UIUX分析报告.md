# AI英语词库 — UI/UX 效果图分析报告

> **分析对象**：26张移动端效果图（iPhone X/11/12/13/14 Pro 尺寸，1125×2448px @3x）  
> **分析方法**：文件名语义解析 + 像素级色彩分析（PIL） + 文件大小比对  
> **总体设计风格**：浅色主题（白色/浅灰为主），绿色作为品牌强调色，界面简洁清晰

---

## 全局设计特征

| 特征 | 值 |
|------|-----|
| 画布尺寸 | 1125×2448px（375×812pt，iPhone 3x Retina） |
| 色彩模式 | RGBA（含透明度） |
| 主背景色 | 白色-浅灰系（RGB均值约 240-250） |
| 品牌强调色 | 绿色（#4CAF50 色系） |
| 顶部导航栏 | 浅灰色（RGB 244,244,244），高度约 120px |
| 底部操作栏 | 浅灰-白色（RGB 240,240,240），高度约 100px |
| 状态反馈色 | 绿色=正确，红色=错误（0.2%-0.7% 红色像素） |
| 结束页特征 | 整体偏灰暗（avg 204-208），可能是全屏插画 |

## 模块二：中译英（词）— 直选模式（图 03-06）

  - 底部：可能是"查看提示"按钮

### 图片 04 — 中译英(词)-直选-答对（103.5KB）
- **像素特征**：白底 78%，绿色 5.4%，avg RGB (230,243,236)；底部绿色 16.2%，红色 0%
- **颜色变化**：相比初始态（95%白→78%白），绿色显著增加
- **界面推断**：
  - 正确选项显示绿色高亮（绿色背景/边框）

  - 底部出现绿色反馈区域（16.2% 绿色像素）
  - 可能显示"正确！"文字 + "下一题"按钮

### 图片 05 — 中译英(词)-直选-答错（103.6KB）
- **像素特征**：白底 71%，绿色 5.4%，红色 0.1%；底部绿色 16.2%，红色 0.2%
- **与答对差异**：白底更少（71% vs 78%），出现微量红色（0.2%），底部 avg 略微偏灰
- **界面推断**：
  - 错误选项显示红色标记（红色边框或 X 图标）
  - 正确选项可能以绿色高亮显示（告知正确答案）
  - 底部反馈区域保持绿色但有红色点缀
  - 可能显示"错误！正确答案是：" + "继续"按钮
- **状态变化**：从答题态 → 红绿标记的错误反馈态

### 图片 06 — 中译英(词)-直选-查看答案（103.5KB）
- **像素特征**：与 04（答对）几乎完全一致（白底 78%，绿色 5.4%）
- **界面推断**：
  - 可能是答错后点击"查看答案"的弹窗/详情页
  - 或与答对状态共用同一反馈界面
  - 显示正确答案、释义、音标等详细信息
- **关联关系**：05（答错）→ 06（查看答案），06 ≈ 04（反馈UI可能复用）

### 用户流程
```
03（看中文选英文）→ 点击选项
  ├─ 正确 → 04（绿色反馈 + 下一题）
  └─ 错误 → 05（红色标记 + 绿色正确答案）→ 06（查看答案详情）→ 下一题
```

## 模块七：中译英（词）— 全拼模式（图 16-18）

### 图片 16 — 中译英(词)-全拼 - 初始（205.7KB，最大之一）
- **像素特征**：白底 83%（比直选94%低），无绿色，avg RGB (243,243,244)
- **文件巨大原因分析**：可能是包含虚拟键盘UI，或复杂布局
  - 底部1/4区域：深色 1.1%，灰色 15.8% — 有一些灰调但不深
  - 推测包含：拼写输入框（字母槽位）+ 26键虚拟键盘
- **界面推断**：
  - 顶部：进度条 + 中文词汇提示
  - 中央上方：字母槽位（每个字母一个格，类似 Wordle/填字游戏）
  - 中央下方：自定义字母键盘（26个英文字母按钮）
  - 键盘布局：QWERTY 排列
- **交互元素**：
  - 字母槽位：显示已输入字母
  - 键盘按钮：点击输入字母
  - 退格键、提交键
  - 可能含"提示"按钮
- **难度**：最高级别，需要完全拼写出英文单词

### 图片 17 — 中译英(词)-全拼-拼对（58.3KB）
- **像素特征**：白底 89%，绿色 5.5%，底部绿色 16.2%，红色 0%
- **界面推断**：
  - 所有字母槽位填满正确字母（绿色高亮）
  - 绿色背景的字母格
  - "拼写正确！"反馈
  - "下一题"按钮
- **文件大小缩小**：从 205.7KB → 58.3KB — 答对后键盘消失，界面简化

### 图片 18 — 中译英(词)-全拼-拼错（58.3KB）
- **像素特征**：白底 89%，绿色 5.4%，红色 0.2%，底部绿色 16.2%，红色 0%
  - **与 17 几乎相同**：仅在顶部有微弱红色差异（avg 相同，绿色相同）
- **推断**：拼对和拼错的视觉差异极小
  - 可能错误字母显示红色，正确部分绿色
  - 底部反馈区域颜色相同
  - 或者拼错后显示正确答案覆盖

### 用户流程
```
16（看到中文提示 + 键盘）→ 逐个输入字母
  ├─ 全对 → 17（所有字母绿色 + 下一题）
  └─ 有错 → 18（错误字母标记红色 + 正确拼写展示）→ 下一题
```

## 设计系统总结

### 色彩体系
| 用途 | 颜色 | RGB范围 | 占比特征 |
|------|------|---------|----------|
| 背景 | 白色/浅灰 | (240-250) | 83-95%（初始态） |
| 强调/成功 | 绿色系 | G > R+50, G > B+50 | 5.4-5.6%（答对态） |
| 底部反馈区 | 浅绿 | (204-243, 237-243, 221-228) | 16.2-16.3%（反馈态） |
| 错误标记 | 红色系 | R > G+50, R > B+50 | 0.1-0.7%（答错态） |
| 结束页背景 | 灰绿色 | (202-208, 210-212, 204-207) | 14-16% 白色占比 |

### UI组件库（推断）
1. **顶部导航栏**：统一高度 ~120px，浅灰背景 (244,244,244)
2. **进度条**：顶部导航栏下方，绿色填充
3. **选项按钮**：白色/浅灰圆角矩形，选中态绿色边框/背景
4. **文本输入框**：白色底 + 底部边框
5. **字母槽位**：方格布局（全拼模式特有）
6. **键盘区**：QWERTY 定制键盘（全拼模式）
7. **反馈弹层**：底部滑入式绿色反馈区（~1/3 屏幕高度，含 16% 绿色）
8. **闪卡**：前后双面设计（新学模式）
9. **插画页**：全屏灰绿色插画（结束态）
10. **底部标签栏**：高度 ~100px，白色背景

### 状态切换逻辑
```
初始态（94-95% 白，0% 绿）
  → 答对：78-83% 白，5.4-5.6% 绿（绿色反馈覆盖）
  → 答错：71-82% 白，5.4% 绿，0.1-0.7% 红（红绿双重标记）
  
答错态
  → 查看答案：同答对反馈UI
  
结束态
  → 14-16% 白，2.7-5.0% 绿（全屏插画覆盖白色主背景）
```

### 反馈UI复用规律
- **单词直选答对**、**全拼答对**、**听力词答对** 的反馈UI参数相同：78% 白 + 5.4% 绿
- **句子类答对**（英译中句、听力句）使用统一模板：80% 白 + 5.4% 绿
- **底部反馈区**在所有答对/答错态中统一：16.2-16.3% 绿色像素
- 红色差异是区分答对/答错的关键：答错比答对多 0.1-0.7% 红色像素

## 交互设计亮点

1. **渐进式难度**：直选（单选）→ 全拼（填空）→ 拼读（拼写+语音），难度递增
2. **多感官学习**：视觉（文字）+ 听觉（发音/听力）+ 动觉（拼写输入）
3. **即时反馈**：每次作答后立即显示正确/错误状态，绿色/红色区分
4. **反馈模板复用**：不同模式的答对/答错使用统一的反馈组件，保持体验一致
5. **新学-练习-复习闭环**：新学 → 练习 → 答错回流到复习 → 复习完成，形成学习闭环
6. **结束页设计差异化**：新学和复习的结束页使用全屏插画（暗色调），与练习页（亮色调）形成鲜明对比
